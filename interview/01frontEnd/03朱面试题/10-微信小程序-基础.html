# 01-简要介绍：和网页的区别

了解：微信小程序 和 网页程序 的 3 个主要区别

运行环境

  • 网页：在浏览器中运行
  • 小程序：在微信中运行

开发环境

  • 网页：代码编辑器 + 网页浏览器(vscode)
  • 小程序：代码编辑器 + 微信模拟器 (微信开发者工具)

调用的 API

  • 网页：BOM API 和 DOM API (document.getElementById(), document.createElement())
  • 小程序：小程序专用 API （网络请求、地理定位、扫码、支付、摄像头、蓝牙、微信登录等） 学习特别多的方法完成固定的功能(查文档)

# 12-视图布局：屏幕适配

通过使用 rpx，让小程序界面能在不同屏幕上同比例缩放

概念介绍

rpx 是一个相对长度单位，会根据当前屏幕尺寸，自动调节实际大小。

一句话记住 rpx：

“所有屏幕宽度都看作 750rpx“

操作步骤

目标：演示 %、rpx、px 三个单位的对比效果

  1. 在界面上放置3个view
<view class="box1">使用百分比</view>
<view class="box2">使用rpx</view>
<view class="box3">使用px</view>
  1. 设置第一个view的宽度为50%
.box1 {
  width: 50%;
  background-color: pink;
}

.box2 {
  width: 375rpx;
  background-color: #88ddff;
}

.box3 {
  width: 187.5px;
  background-color: teal;
}
  1. 在模拟器中选择不同设备机型，观察3个view的长度变化
  • 使用 rpx 修正轮播图案例的尺寸，使它在任何设备下看起来都一致
/* 
  如何将设计稿的px换成rpx单位 
    1. 整个屏幕就是750rpx = 设计稿宽度375px
      => 1px = 2rpx
    2. 设计稿中的160px 就是320rpx
*/
swiper {
  width: 750rpx;
  height: 320rpx;
}

image {
  width: 100%;
  height: 100%;
}

本章回顾

  • Q1：如果设计稿宽度是375px，那设计稿上标的1px用rpx来表示是多少？
1. 将750除以设计稿的宽度，以此算出倍率。（750/375 = 2）
2. 将设计稿上的尺寸乘以倍率。（1 * 2 = 2rpx）

# 14-WXS：基本用法

通过小示例了解 WXS 的用法：使用 WXS 函数来格式化日期时间戳

WXS 与 JS 的语法主要差异（更多语法参考 官方文档open in new window）：

  • 变量声明只能用 var
  • 变量名不能有 $
  • 不能用关键字 new

  • 要使用 <wxs>标签，并且必须设置好 module 属性

操作步骤

  1. 在页面js中准备一个日期时间戳数据

  }

<view class="now">{{ now }}</view>

<!-- wxs 标签必须指定 module 属性 -->
<wxs module="formatTools">
  function formatDate(timestamp) {
    // 通过 getDate() 将时间戳转换成日期对象
    var d = getDate(timestamp);
  
    // 拼接成可阅读的格式，如：2020年10月10日
    return d.getFullYear() + '年' + (d.getMonth() + 1) + '月' d.getDate() + '日';
  }

  module.exports = {
 	 	formatDate: formatDate
  }
</wxs>

or

wxs/format.wxs

function formatDate(time) {
  // new Date(time)
  var date = getDate(time)
  var year = date.getFullYear()
  var month = date.getMonth() + 1
  var day = date.getDate()

  return year + '年' + month + '月' + day + '日'
}

module.exports = {

Vant Weappopen in new window 是移动端 Vue 组件库 Vantopen in new window 的小程序版。

两者基于相同的视觉规范，提供一致的 API 接口，帮助开发者能快速的搭建小程序应用。

操作步骤

  1. 按照小程序中使用 npm 包的方式安装 Vant Weapp
yarn add @vant/weapp --production

工具 => 构建npm

  1. 全局注册 Vant Weapp 中的组件
"usingComponents": {
  "van-button": "@vant/weapp/button/index"
}
  1. 调用 Vant Weapp 组件
<van-button type="primary">按钮</van-button>
