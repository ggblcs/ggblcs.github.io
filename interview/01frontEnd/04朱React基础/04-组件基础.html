# 04 -组件基础

React 中存在两种编写组件的方式：函数组件(没有自己的数据的)、类组件(维护自己的状态)

react v16.8.0 提出了hooks

# 创建函数组件

**目标：**使用函数创建一个组件

创建函数组件时需遵循一些约定：

  • 函数名必须以大写字母开头，因为 React 以此区分是 组件 还是 普通函数
  • 函数返回值是 React 元素，代表组件的界面
  1. 创建组件
// 使用普通函数创建组件：
function Hello() {
  return <div>这是我的第一个函数组件!</div>
}

// 使用箭头函数创建组件：
const Hello = () => {
  return <div>这是我的第一个函数组件!</div>
}
  1. 调用组件：使用标签形式来调用组件函数，而不是普通的函数调用形式
// 使用 双标签
ReactDOM.render(<Hello></Hello>, root)

// 使用 单标签
ReactDOM.render(<Hello />, root)

小结：

  1. 可以使用普通函数或箭头函数创建 React 组件
  2. 组件函数的首字母必须大写

# 创建类组件

目标：使用 ES6 的 class 语法创建组件

创建类组件时需要遵循的一些约定：

  • 类名必须以大写字母开头
  • 需要继承父类 React.Component
  • 类中必须有一个 render() 方法，且它的返回值为 React 元素

核心代码

  1. 定义组件：
// 导入 React
import React from 'react'

class Hello extends React.Component {
  render() {
    return <div>Hello Class Component!</div> 
  }
}

ReactDOM.render(<Hello />, root)
  1. 调用组件：
ReactDOM.render(<Hello />, document.getElementById('root'))

总结

  1. 使用 class 语法来创建组件，看起来要比函数的创建稍麻烦，但类组件的功能会比函数组件更强大（但注意：hooks 出现之后就不存在这样的差异了）

# 将组件放入单独文件

目标：将组件代码写到一个独立的 js 文件中

每个组件都应该写到一个独立的 .js/.jsx 文件中，这样便于管理。

实施步骤：

  1. 创建 Hello.js
  2. 创建组件（函数组件 或 类组件都可）
  3. 在 Hello.js 中导出该组件
  4. 在 index.js 中导入 Hello 组件并调用

核心代码

// Hello.js
import { Component } from 'react'

class Hello extends Component {
  render() {
    return <div>Hello Class Component!</div>
  }
}

// 导出Hello组件
export default Hello

// index.js
import Hello from './Hello'

// 渲染导入的Hello组件
ReactDOM.render(<Hello />, root)

# 有状态组件和无状态组件

目标：理解函数组件和类组件的区别

什么是状态？

状态，是指组件内的私有数据（和 Vue 组件中的 data 一样）。当状态发生变化时，用到该状态的页面结构会随之发生更新（数据驱动视图）。

比如下列计数器，当用户点击按钮后，数值会进行+1。这里的 0 和 1 其实就是该组件内的一个不断变化的状态，当它从 0 变成 1 后，界面也跟着一起发生变化：

<img src="images/image-20211007121657774.png" alt="image-20211007121657774" style="zoom:50%;" />

无状态组件：

  • 在 hooks 技术出现之前，函数组件属于 无状态组件，因为它内部不能维持私有数据，只能靠外部传入数据
  • 无状态组件的作用：只负责从外部接收数据，并渲染界面结构（逻辑比较简单，因此性能较高）

有状态组件：

  • 类组件、以及出现 hooks 技术后的函数组件，都属于 有状态组件，它们内部可以维持私有数据，且当这些数据发生改变后会重新渲染界面
  • 实际开发中大多数组件都需要拥有内部私有数据，所以几乎所有组件都被写成有状态组件

# 类组件：事件处理-this指向问题

目标：了解事件处理方法中 this 指向 undefined 的现象，并了解原因

测试一下：

  • 打印事件处理方法中的 this
  • 打印 render 或其他方法中的 this

测试结果：

  • 事件处理方法中的 this => undefined
  • render 及其他方法中的 this => 当前组件实例
class App extends React.Component {
  state = {
    msg: 'hello react'
  }

  handleClick() {
    console.log(this.state.msg)
  }

  render() {
    return (
      <button onClick={this.handleClick}>点我</button>
    )
  }
}

发生这种现象，其实是 Babel 在转换 JSX 时，生成的代码变成了如下形式：

// 1. 严格模式
'use strict';

class App extends React.Component {
  state = {
    msg: 'hello react'
  }

  handleClick() {
    console.log(this.state.msg)
  }

  render() {
    // 2. 变为了普通函数
    const fn = this.handleClick
    
    return React.createElement('button', { 
      onClick: fn 
    }, '点我')
  }
}

  1. JSX 标签上的事件处理方法被设到了一个变量中，成为一个“普通函数” 而不是对象方法。所以在调用它时，调用者已不是原来的组件实例，而是 window

  2. 另外，由于转换后的代码被设为 JS “严格模式” ，在该模式下，普通函数中的 this 不再是 window 而是 undefined

总结：

  1. 在 React 的事件处理函数中，this 指向 undefined

# 类组件：事件处理-解决this指向

**目标：**掌握常见的 this 指向解决方案

方式1：使用内嵌箭头函数

class App extends React.Component {
  state = {
    msg: 'hello react'
  }

  handleClick() {
    console.log(this.state.msg)
  }

  render() {
    return (
      <button onClick={() => { this.handleClick()}}>点我</button>
    )
  }
}

相当于转换后：

render() {
  const fn = () => { 
    this.handleClick()
  }
  
  // ...
}

方式2：使用 bind 强制绑定 this

class App extends React.Component {
  state = {
    msg: 'hello react'
  }

  handleClick() {
    console.log(this.state.msg)
  }

  render() {
    return (
      <button onClick={this.handleClick.bind(this)}>点我</button>
    )
  }
}

相当于转换后：

render() {
  const fn = this.handleClick.bind(this)
  
  // ...
}

方式3：将方法改写为属性形式

class App extends React.Component {
  state = {
    msg: 'hello react'
  }

  handleClick = () => {
    console.log(this.state.msg)
  }
  
  render() {
    return (
      <div>
        <button onClick={this.handleClick}>点我</button>
      </div>
    )
  }
}

相当于转换后：

handleClick = () => {
  console.log(this.state.msg)
}

render() {
  const fn = this.handleClick  // this.handlerClick 是个箭头函数

  // ...
}

总结

  1. 方式 3 是企业中最常用的方式

# 类组件：修改状态

**目标：**掌握在类组件中通过 setState 方法来修改状态

语法：this.setState({ 要修改的状态 })

说明：修改状态后，组件会发生重新渲染

容易犯的错误：

如果直接修改 state 对象上的值，比如：this.state.xxx = 11111，这样也能改掉数据，但是，它不会触发视图的更新，会造成数据和视图不同步的问题！

所以一定要用 setState() 方法来修改状态！

核心代码

class Hello extends Component {
  state = {
    count: 0
  }

  render() {
    return (
      <div>
        <h1>计数器：{this.state.count}</h1>
        <button onClick={() => {
          this.setState({ 
            count: this.state.count + 1 
          })
        }}>增加</button>
      </div>
    )
  }
}

# 最佳实践：修改状态的正确写法

**目标：**掌握 React 组件的标准状态修改写法

遵守规则：

永远不要去动旧的状态，而应该总是创建一个新状态，去替换旧状态

比如，当前有如下状态：

// 当前的状态
state = {
  count: 0,
  item: {
    name: 'Apple',
    price: 8.8
  }
}

❎ 错误的更新方式：

// 直接修改当前状态对象中的值
this.state.count = 1
this.state.item.price = 9.9

// 然后用 setState 进行更新
this.setState(this.state)

⚠️ 注意：这么做虽然也会有效果，但是会报出警告

!---image-20220321165451917

✅ 正确的更新方式：

// 创建一个全新的状态对象
const newState = {
  count: 1,
  item: {
    ...this.state.item,
    price: 9.9
  }
}

// 然后用 setState 进行更新
this.setState(newState)

# 类组件：表单处理-受控组件

**目标：**能使用受控组件的方式，获取文本框中输入的内容

受控是什么意思？

指表单组件的数据只受状态 ( state ) 的控制！

具体表现为：在受控方式下，表单组件中显示的内容只会随着与它绑定的 state 值的改变而改变，而不会直接受用户手动输入的影响！

具体用法：

  1. 先将表单组件的 value/checked 属性与 state 关联

  2. 再为表单组件设置 onChange 事件监听，并在获取最新输入内容后更新到 state 中

核心代码：

class App extends React.Component {
  state = {
    msg: 'hello react'
  }

  handleChange = (e) => {
    this.setState({
      msg: e.target.value
    })
  }

  render() {
    return (
      <div>
        <input type="text" value={this.state.msg} onChange={this.handleChange}/>
      </div>
    )
  }
}

总结：

所以，使用受控组件的方式处理表单后，状态的值就是表单元素的值。

想要操作表单的值，只需操作对应的状态即可。

# 类组件：表单处理-非受控组件

**目标：**使用操作原生 DOM 的方式操作表单元素

  • 受控组件 是通过 React 组件的状态来控制表单元素的

  • **非受控组件 **是通过直接操作 DOM 的方式来控制表单元素

实现方法：通过 ref 获取渲染后的真实 DOM。

核心代码

// 引入 createRef 方法
import { createRef } from 'react'

class Hello extends Component {
  
  // 创建一个 Ref 对象
  txtRef = createRef()

  handleClick = () => {
    // 获取文本框对应的 DOM 元素
    const el = this.txtRef.current
    // 获取文本框中的值
    const value = this.txtRef.current.value
    console.log(value)
  }

  render() {
    return (
      <div>
        <input type="text" ref={this.txtRef} />
        <button onClick={this.handleClick}>获取文本框的值</button>
      </div>
    )
  }
}

# 筛选排序

  • 修改active 和 list属性

changeTab = (type) => {
  let newList = [...this.state.list]
  
  if (type === 'hot') {
    // 按照id进行排序
    newList.sort((a, b) => a.id - b.id)
  } else {
    newList.sort((a, b) => a.time - b.time)
  }
  
  this.setState({
    active: type,
    list: newList
  })
}
