# 组件通信介绍

目标：了解为什么需要组件通信？

组件为什么需要通信？

组件只是完整界面的一部分，多个组件要互相合作才能形成完整功能。

而组件合作的方式，其实就是互相传递工作所需的数据，所以它们需要一些传递数据的手段。

  • 默认情况下，组件是封闭的，状态数据只能在组件内部使用

  • 如果组件要和其他组件通信，就必须在封闭的组件上打开一些缺口，让内部和外部的数据能进出

# props：React 最重要的组件通信机制

**目标：**了解组件用于接收外部数据的机制 props

概念

props（properties 的缩写），即 “属性” 的意思。通过它可以让组件接收外部数据。

用法

传递数据到组件 props 中：

  1. 方式一：给组件标签设置属性，如：<Comp title="xxxx" />

  2. 方式二：给组件标签中间设置内容，如：<Comp>hello</Comp>

从组件 props 中获取数据：

  • 在函数组件中的获取方式： function Comp(props) {}
  • 在类组件中的获取方式： constructor(props) 和 this.props

核心代码：

  1. 在函数组件中获取 props 的方式

传递数据：

<Hello name="jack" age={19} />

接收数据：

function Hello(props) {
  return (
    <div>接收到数据:{props.name}</div>
  )
}
  1. 在类组件中获取 props 的方式

传递数据：

<Hello name='jack' age={19} />

接收数据：

class Hello extends Component {
  
  // 在构造函数中，通过参数获取
  constructor(props) {
  	super(props)
    console.log(props)
  }
  
  // 在方法中，通过 this.props 获取
  render() {
    return (
      <div>接收到的数据:{this.props.age}</div>
    ) 
  }
}

# 使用 props 时的注意事项

目标： 了解 props 的一些特点，并知道什么叫 “单向数据流”

内容：

  1. 组件的 props 是一个只读对象，只能读取它的值，而不要尝试去修改它

这是一种被称为 单向数据流 的设计方式（也叫：自上而下的数据流）。

它表示：父组件中的数据能通过 props 传递给子组件，且当父组件的数据发生变化时，子组件也会自动收到最新数据。但该过程不能反过来，即让子组件去修改父组件的数据。

【比喻：瀑布的水只能从上往下流，不能反过来。且当上游水变浑时，下游水也会受影响，但下游如果有什么变化，不会影响上游】

  1. React 组件的 props 能传递任意类型的数据，比如：基础类型、对象类型、函数、JSX

  • 如果要传递的值是字符串，可这样写： title="bb" 或 title={"bb"}

  • 如果要传递的是其他类型，则一定要放在大括号中： num={100}

推荐做法：不管什么情况总是统一用大括号 {}

# 组件通讯-父传子

目标：将父组件中的数据传给子组件

操作步骤：

  1. 首先，在父组件中提供要被传递到子组件的 state 状态

  2. 然后，将要被传递的 state 状态设置到子组件的标签属性上

  3. 最后，在子组件中通过 props 接收父组件传来的状态

通过父传子实现以下功能：

核心代码

父组件提供数据，并传递给子组件：

class Parent extends React.Component {
  state = { 
    money: 10000 
  }

  render() {
    return (
      <div>
        传递数据给子组件：<Child money={this.state.money} />
      </div>
    )
  }
}

在子组件中接收父组件传来的数据：

function Child(props) {
	return <div>子组件接收到数据：{props.money}</div>
}

# 组件通讯-子传父

**目标：**能够将子组件的数据，传给它的父组件

操作步骤：

  1. 让父组件为子组件传入一个值为函数的 props
  2. 在子组件中获取并调用该函数，并将子组件数据作为参数传出。

通过子传父实现以下功能：

在之前的案例中的子组件里，放一个砍价按钮，将点击砍价按钮后的新价格传给父组件：

核心代码：

父组件提供回调函数，并且传递给子组件

class Parent extends React.Component {
  getChildMsg = (msg) => {
    console.log('接收到子组件数据', msg)
  }

  render() {
    return (
      <div>
        子组件：<Child getMsg={this.getChildMsg} />
      </div>
    )
  }
}

在子组件中获取回调函数，并调用

class Child extends React.Component {
  state = { childMsg: 'React' }

  handleClick = () => {
    // 获取回调函数并调用，传出数据
    this.props.getMsg(this.state.childMsg)
  }

  return (
  	<button onClick={this.handleClick}>
      点我，给父组件传递数据
    </button>
  )
}

# 组件通讯-兄弟间传值

**目标：**能理解什么叫状态提升，并用它来实现兄弟组件之间的数据传递

内容：

状态提升，就是指将需要共享的状态，放到离得最近的公共父组件中，由该公共父组件管理该状态

公共父组件的职责：

  • 提供共享状态、以及操作共享状态的方法

  • 需要互相通讯的子组件们，只需要和公共父组件进行常规的父传子、子传父即可

状态提升之后：

核心代码

index.js

import React, { Component } from 'react'
import ReactDOM from 'react-dom'
import Jack from './Jack'
import Rose from './Rose'

class App extends Component {
  // 1. 状态提升到父组件
  state = {
    msg: '',
  }
  
  render() {
    return (
      <div>
        <h1>我是App组件</h1>
        <Jack say={this.changeMsg}></Jack>
        {/* 2. 把状态给子组件显示 */}
        <Rose msg={this.state.msg}></Rose>
      </div>
    )
  }
  
  changeMsg = (msg) => {
    this.setState({
      msg,
    })
  }
}

// 渲染组件
ReactDOM.render(<App />, document.getElementById('root'))

# 组件通讯 - Context

**目标：**通过 React 的 Context 机制实现跨级组件间的通讯（也包括父子、兄弟）

跨级组件通信：逐层传递方式

!---props层层传递

更好的方式？

!---跨组件传递

逐层方式实现起来麻烦，组件耦合度高。我们可以采用的更好的方式是：使用 React 自带的 Context 机制。Context 即上下文， 可理解为是一个范围，只要在该范围内的组件可以直接互相通讯。

核心代码

import { createContext } from 'react'

// 1 创建 Context 对象
const { Provider, Consumer } = createContext()

// 2 使用 Provider 组件包裹整个应用，并通过 value 属性提供要共享的数据
<Provider value="blue">
  <div className="App">
    <Child1 /> 
  </div>
</Provider>

// 3 使用 Consumer 组件接收要共享的数据
<Consumer>
  {data => {
    return <span>data参数表示接收到的数据 -- {data}</span>
  }}
</Consumer>

# Tab 栏点击切换

**目标：**完成tab栏切换功能

步骤

  1. 父组件将 tabs, active 两个状态数据传给 Tabs 子组件
  2. 在 Tabs 组件中通过 props 接收传入的 tabs 数据，然后渲染 tab 栏
  3. 点击 tab 栏后，将当前点击的 type 传给父组件

核心代码

  • 父组件传值给子组件
const { tabs, active } = this.state

{/* 排序 */}
<Tab tabs={tabs} active={active}></Tab>
  • 子组件接收并且渲染
const { tabs, active } = this.props

<ul className="sort-container">
  {tabs.map((item) => (
    <li className={item.type === active ? 'on' : ''} key={item.id}>
      按{item.name}排序
    </li>
  ))}
</ul>
  • 父组件提供 changeTab
changeTab = (type) => {
  this.setState({
    active: type,
  })
}
  • 父组件将changeTab传递给子组件
<Tab tabs={tabs} active={active} changeTab={this.changeTab}></Tab>
  • 子组件接收changeTab并且注册事件进行切换
render() {
  const { tabs, active, changeTab } = this.props
  return (
    <div className="tabs-order">
      <ul className="sort-container">
        {tabs.map((item) => (
          <li
            onClick={() => changeTab(item.type)}
            className={item.type === active ? 'on' : ''}
            key={item.id}
          >
            按{item.name}排序
          </li>
        ))}
      </ul>
    </div>
  )
}

onAdd = (content) => {

import React, { Component } from 'react'

    return (
